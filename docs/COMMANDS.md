# AegisRAG — Full Command Reference

Every command you need, in the order you need it. Copy-pasteable.

> **Current setup snapshot (as of April 2026)**
> - Environment: `conda activate dl` (Python 3.10)
> - Device: Apple Silicon Mac (MPS detected, CPU used for GGUF inference)
> - Generator: `checkpoints/aegis_dpo.gguf` (Qwen2.5-7B q4_k_m, 4.4 GB) — **already built**
> - Reranker: `cross-encoder/ms-marco-MiniLM-L-6-v2` fine-tuned checkpoint at `checkpoints/reranker/`
> - Retriever: `BAAI/bge-m3` fine-tuned checkpoint at `checkpoints/retriever/`
> - Training: No SFT, no decomposer — pipeline is retriever → reranker → DPO → confidence → alpha
> - Data: 500 QA pairs, 200 preference pairs (rule-based, no API)
> - API prefix: `/api/v1` (e.g. `http://localhost:8000/api/v1/query`)

## 2. Document ingestion (build the knowledge base)

```bash
# 2.1 Drop source docs into data/raw_docs/ — PDFs, DOCX, TXT, Markdown, CSV, JSON
ls data/raw_docs/

# 2.2 Ingest: parse → chunk (256 tokens, 96 overlap) → MinHash dedup → ChromaDB + BM25
conda activate dl
python run.py ingest --source-dir data/raw_docs

# 2.3 Verify chunk count
python -c "
from src.retrieval.vector_store import ChromaVectorStore
store = ChromaVectorStore()
print(f'Chunks indexed: {store.collection.count()}')
"
```

> Vector DB lives at `data/vectordb/`, BM25 index at `data/bm25_index.pkl`.
> Re-ingesting is safe — MinHash dedup (threshold 0.92) prevents duplicates.

## 3. Synthetic training data generation (local, no APIs)

All data is generated by `LocalTeacher` (Qwen2.5-7B loaded from HF cache) with no external API calls.

```bash
conda activate dl

# 3.1 QA pairs only (500 pairs, ~1-2h on CPU)
python run.py generate-data --type qa --output-dir data/synthetic

# 3.2 Preference pairs only (200 pairs, rule-based, ~5 min)
python run.py generate-data --type preference --output-dir data/synthetic

# 3.3 Confidence labels (500 soft-label examples)
python run.py generate-data --type confidence --output-dir data/synthetic

# 3.4 Alpha network labels (500 oracle fusion weights)
python run.py generate-data --type alpha --output-dir data/synthetic

# 3.5 Or everything at once
python run.py generate-data --type all --output-dir data/synthetic
```

> `decomp` type is skipped — the decomposer is rule-based (spaCy), no training data needed.

Output files in `data/synthetic/`:
- `qa_pairs.jsonl` — 500 query/answer/chunk triples
- `preference_pairs.jsonl` — 200 chosen/rejected DPO pairs (5 rejection types)
- `confidence_labels.jsonl` — soft-label confidence scores
- `alpha_labels.jsonl` — oracle alpha fusion weights

## 4. Training

**Already trained and converted to GGUF. Skip to Section 7 if `checkpoints/aegis_dpo.gguf` exists.**

Training order (retriever → reranker → dpo → confidence → alpha):

```bash
conda activate dl

# 4.1 All components in correct order
python run.py train --component all

# Or individually:

# 4.2 Retriever (BGE-m3, InfoNCE loss, ~1-2h CPU)
python run.py train --component retriever
# Output: checkpoints/retriever/

# 4.3 Reranker (cross-encoder/ms-marco-MiniLM-L-6-v2, BCE, ~20-30 min CPU)
python run.py train --component reranker
# Output: checkpoints/reranker/model.pt + tokenizer files

# 4.4 DPO alignment (DoRA rank-8, IPO loss, 200 pairs, ~1-2h CPU)
python run.py train --component dpo
# Output: checkpoints/dpo/

# 4.5 Confidence head (MSE loss, NLI-entailment soft labels, ~10 min)
python run.py train --component confidence
# Output: checkpoints/confidence_head/

# 4.6 Alpha fusion network (tiny MLP, MSE, ~2 min)
python run.py train --component alpha
# Output: checkpoints/alpha_network/
```

> **SFT is skipped** — no `generator` or `decomposer` component.
> Running `python run.py train --component generator` will raise a clear error by design.

## 7. Running the system (post-GGUF — start here)

### 7.1 Single query (no server needed)

```bash
conda activate dl

# Non-streaming
python run.py query --model m5 --query "How do I reset my password?"

# Streaming (token-by-token output)
python run.py query --model m5 --query "What documents do I need to file for an LLC?" --stream
```

### 7.2 FastAPI server

```bash
conda activate dl

# Start the server (GGUF auto-detected, loads in ~10s)
python run.py serve --model m5 --host 0.0.0.0 --port 8000
```

**API prefix is `/api/v1` — all routes live under it:**

```bash
# Health check
curl http://localhost:8000/api/v1/health

# Single query (JSON response)
curl -s -X POST http://localhost:8000/api/v1/query \
  -H "Content-Type: application/json" \
  -d '{"query": "How do I reset my password?", "model_tag": "m5"}' | python -m json.tool

# Streaming query (SSE)
curl -N -X POST http://localhost:8000/api/v1/query/stream \
  -H "Content-Type: application/json" \
  -d '{"query": "What is the refund policy?", "model_tag": "m5"}'

# Baseline comparison (b1/b2/b3)
curl -s -X POST "http://localhost:8000/api/v1/query/baseline?baseline=b3" \
  -H "Content-Type: application/json" \
  -d '{"query": "How do I file a complaint?", "model_tag": "b3"}' | python -m json.tool

# Prometheus metrics
curl http://localhost:8000/api/v1/metrics

# Escalation tickets
curl http://localhost:8000/api/v1/tickets

# Upload new documents at runtime
curl -X POST http://localhost:8000/api/v1/ingest \
  -F "files=@data/raw_docs/my_policy.pdf"
```

## 8. What runs where (Apple Silicon specifics)

| Component | Backend | Device | Notes |
|-----------|---------|--------|-------|
| BGE-m3 retriever | HuggingFace transformers | MPS | fp16, fast |
| ColBERT reranker | HuggingFace transformers | MPS | fp32 fallback |
| Qwen2.5-7B generator | llama-cpp-python (GGUF) | CPU | `n_gpu_layers=0` — MPS crashes on quantized attention |
| Confidence head | PyTorch | MPS | tiny MLP |
| Alpha network | PyTorch | MPS | tiny MLP |
| NLI verifier | HuggingFace transformers | MPS | cross-encoder |

To force CPU for everything (useful for debugging):
```bash
AEGIS_DEVICE='{"preferred_device":"cpu"}' python run.py query --model m5 --query "test"
```

## 9. Troubleshooting cheatsheet

| Symptom | Likely cause | Fix |
|---|---|---|
| `zsh: command not found: python` | conda env not active | `conda activate dl` |
| `'dict' object has no attribute 'paths'` | config passed as raw dict | already fixed in `run.py` — update from repo |
| `QueryRequest is not defined` | Pydantic v2 forward ref in local class | already fixed in `app.py` — update from repo |
| MPS matmul abort / Metal crash | GGUF with `n_gpu_layers > 0` on MPS | `gguf_n_gpu_layers: 0` in `config/base.yaml` (already set) |
| HF model loads instead of GGUF | old generator.py backend selection bug | already fixed — checks `cfg.models.generator.gguf_path` |
| `404 Not Found` on `/query` | missing `/api/v1` prefix | use `http://localhost:8000/api/v1/query` |
| `chromadb.errors.InvalidCollectionException` | stale DB from old version | `rm -rf data/vectordb && python run.py ingest ...` |
| Confidence always escalates | thresholds above model output range | thresholds already lowered to 0.55/0.40/0.20 in `config/base.yaml` |
| Answer truncates mid-sentence | `max_new_tokens` hardcoded to 256 | already fixed — reads 512 from config |
| `ForwardRef('List[UploadFile]')` error | `from __future__ import annotations` in ingest_router | already fixed — import removed |
| `run.py train --component generator` | SFT intentionally skipped | use `--component dpo` after retriever+reranker |
| Slow first query (~30s) | lazy GGUF load | run one warmup: `python run.py query --model m5 --query "warmup"` |

## 10. Utility & maintenance

```bash
conda activate dl

# 10.1 Run all tests
pytest tests/ -v

# 10.2 Tail the audit log (queries, confidence, latency)
sqlite3 data/audit.db "SELECT timestamp, model_tag, confidence, latency_ms FROM queries ORDER BY timestamp DESC LIMIT 20;"
```
